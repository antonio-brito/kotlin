class com.globalkit.microapps.bus.service.Bus

Main object-class which provides bus functionality. You must use this object for subscribing to any topic , publishing messages to any topic and unsubscribing from a topic. This implementation follows a Publish-Subscribe pattern, which is a messaging pattern, following a topic-based filtering.

This means:

  • if you want to attend requests, at a specific topic, you must subscribe to that specific topic through subscribe method
  • if you want to publish a message, to a specific topic, you must send a RequestMessage instance to that specific topic through publishAsync method
  • if you want to stop attending requests, you must unsubscribe through unsubscribe method

fun subscribe(topic: String, function: (RequestMessage)->, ex: (Throwable)->Unit):

This method allows you to subscribe to a specified topic in order to attend messages. You must specify, within a lambda function, how to treat/process the request instance and must return a Single<ResponseMessage>

Parameters:
  • topic: String

    topic you want to subscribe to

  • function: (RequestMessage)->

    function: (RequestMessage) -> Single<ResponseMessage> this is the lambda function to be invoked/executed when a new message arrives. Within this function, the subscriber indicates how to treat the request and how to generate the response instance. It is highly recommended that the response id, at id attribute, contains the value of the request id in order to indicate request - response relationship

  • ex: (Throwable)->Unit

    exceptional lambda function to be invoked/executed when an exception occurs

Returns:

com.noheltcj.rxcommon.observables.Single<ResponseMessage>

ResponseMessage indicates whether the subscription succeeded or not. It also contains a subscription id, at id attribute, which you must keep in order to unsubscribe later through unsubscribe(subscriptionId: String) method

fun publishAsync(topic: String, message: RequestMessage): ?

This method allows you to publish a message, in a broadcast mode, to a specific topic and get an Observable containing the responses for that message; responses from all subscribers in the specified topic.

Parameters:
  • topic: String

    topic you want to publish into

  • message: RequestMessage

    RequestMessage instance containing the proper information in order to be correctly attended by subscribers

Returns:
?

com.noheltcj.rxcommon.observables.Observable<ResponseMessage>

Each ResponseMessage instance indicates:

  • whether the message was successfully attended or not
  • if there was an error, you could get the exception detail from the error attribute
  • optionally, the response ID, at id attribute. This id will be set by the subscriber which attended the request and generated the response instance. This attribute is mainly intended to specify the request id this response is for
fun unsubscribe(subscriptionId: String):

This method allows you to stop subscribers from attending requests. In performance terms, this is a very important method you must call, once per subscription, whenever you need toindicate that the subscriber is not attending requests anymore

Parameters:
  • subscriptionId: String

    subscriber's subscription id you obtained when you subscribed, through subscribe method, and you want to unsubscribe

Returns:

com.noheltcj.rxcommon.observables.Single<ResponseMessage>

ResponseMessage instance indicates:

  • whether the unsubscribe operation was successfully processed or not
  • if any error occurred, you could get the exception detail from the error attribute

Note: this response will never specify an id; id attribute will always be null

fun publishAsyncWithoutResponse(topic: String, message: RequestMessage):

This method allows you to deliver a message, in a broadcast mode, to a specific topic.

The unique difference with publishAsync(topic: String, message: RequestMessage) method is that this method will not retrieve any responses for the delivered message.

Use publishAsyncWithoutResponse(topic: String, message: RequestMessage) instead of publishAsync(topic: String, message: RequestMessage) method when you do not need any responses from subscribers, you just need to deliver a message and you do not matter what happens after that

Parameters:
  • topic: String

    topic you want to deliver the message into

  • message: RequestMessage

    RequestMessage instance containing the proper information in order to be correctly attended by subscribers

Returns:

com.noheltcj.rxcommon.observables.Single<ResponseMessage>

The ResponseMessage instance indicates:

  • the response ID, at id attribute. This id will be the same value at the provided request instance
  • the result, at result attribute, which will be one of these JSON formatted messages: {'status':'delivered'} or {'status':'not delivered'}

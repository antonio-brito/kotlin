class com.globalkit.microapps.bus.core.domain.RequestMessage

RequestMessage is a data class which is instantiated any time whe need to publish into the Bus. This class indicates the id, headers and parameters for any massage to be passed through the Bus; this class represents the request messages to be passed

Also look at ResponseMessage, Bus

Fields

id: String

required Unique id for a request instance. Every time a new instance of this class is created, a new id must be provided. Te main goal of this attribute is to have a unique id per request in order to get responses, from subscribers, for this id.

headers: String?

optional Json formatted headers needed by subscribers in order to successfully attend the request. This headers are specified by any subscriber, in case it needs any.

parameters: String?

optional Json formatted parameters needed by subscribers in order to successfully attend the request. This parameters are specified by any subscriber, in case it needs any.
